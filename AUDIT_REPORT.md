## 0. Executive Summary (read this first)

The previous AI's commit messages claim many fixes that **do not hold up when you read the actual code**. The app is **not currently deployable in a working state**. Even if you set every environment variable correctly and deploy, the search flow would fail at runtime because the worker writes to database columns that do not exist.

### The 5 things that would block a live deployment right now

1. **Worker cannot persist leads.** `task_worker/worker.py` inserts a `lead_hash` column into `workspace_leads` (the real column is `domain_hash`) and a `task_type` column into `smart_collections` (no such column exists). Every lead insert would fail. The task would be marked "completed" with zero leads stored. (Backend audit, section E.)
2. **Users get charged twice and lose credits on failed searches.** `deduct_coins` is called once on task creation (reserve) and again on task completion (intended as "deduct" but actually reserves again). No refund RPC exists for `failed` or `exhausted` tasks. Locked credits never return. (Backend audit, section F.)
3. **`add_coins` is not idempotent.** A double Paystack webhook would double-credit. No `reference_id` parameter, no dedup check. (Backend audit, section B.)
4. **RLS is disabled on every table.** Anyone with the anon key can read/write any user's data directly through Supabase. (Backend audit, section B.)
5. **Backend auth is optional.** If `BACKEND_API_SECRET` is unset (which is the default), every route is open. CORS is `*`. There is no rate limiting. (Backend audit, section H.)

### The 5 things the previous AI claimed were fixed but are NOT

1. **Pricing consistency** (claimed fixed in commit `9d6b0f8`) — NOT fixed. Free tier unlocks 2 engines (should be 1), `credits-3000` package is missing, no per-tier "credits per lead" concept, and there is a dead duplicate `src/components/pricing.tsx` with its own conflicting data. (Frontend audit, section A.)
2. **Engine locking** (claimed fixed in `9d6b0f8`) — PARTIAL. Locked engines don't run a search (good), but clicking a locked card does nothing instead of opening pricing/upgrade, and the badge says "Pro" when the engines actually unlock at Starter. (Frontend audit, section B.)
3. **Coins → credits rename** — NOT started. 282 "coin" matches in frontend, 84 in backend, 0 "credit" matches. (Frontend audit G, backend audit J.)
4. **Perplexity teal theme** — NOT applied. The old dark theme (`#08080C`, `#7C5CFC`, `#3B82F6`, `#F5F5F7`) is still in `globals.css`, `layout.tsx`, and every page/component. Zero teal color values exist anywhere. (Frontend audit, section H.)
5. **Product name "Bad Decision"** (not "Bad Decision AI") — NOT applied. 18 user-facing "Bad Decision AI" strings remain (page title, nav brands, footer copyright, About page body). (Frontend audit, section I.)

## 4. Critical security findings
